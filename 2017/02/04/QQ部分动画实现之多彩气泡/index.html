前言

本篇是QQ动画实现系列的第二篇，是气泡动画类型的一个小集合，比上篇稍微难一点，不过我们可以逐步分析、循循渐进，跟大家一起学习一下这些动画是如何实现的。

  1. 用户头像像橡皮糖一样上下左右伸缩变形。
  2. 头像背景有气泡向四周消散。

一、实现橡皮糖效果

从第一篇文章《QQ部分动画实现之点赞效果》的学习，我们知道了ValueAnimator的用法。分析这个橡皮糖动画，无非就是分别设置控件的ScaleX属性以及ScaleY属性，就可以实现水平方向以及垂直方向的拉伸效果。反复观察这个动画，发现变形是跟球在地上弹跳一样，第一下弹得高，第二下弹得低一些，低三下更低直到不再弹跳为止。因此这里我们需要用到插值器Interpolator，它的作用是用来指定动画如何变化。官方已经提供了一些插值器给我们使用，当然我们也可以自定义。因为这个弹跳插值器已经有提供，所以我们直接使用即可。其他官方提供的插值器如下：

意义如下：

  • AccelerateDecelerateInterpolator 在动画开始与介绍的地方速率改变比较慢，在中间的时候加速
  • AccelerateInterpolator 在动画开始的地方速率改变比较慢，然后开始加速
  • AnticipateInterpolator 开始的时候向后然后向前甩
  • AnticipateOvershootInterpolator 开始的时候向后然后向前甩一定值后返回最后的值
  • BounceInterpolator 动画结束的时候弹起
  • CycleInterpolator 动画循环播放特定的次数，速率改变沿着正弦曲线
  • DecelerateInterpolator 在动画开始的地方快然后慢
  • LinearInterpolator 以常量速率改变
  • OvershootInterpolator 向前甩一定值后再回到原来位置

二、实现气泡向四周消散的效果

其实这个动画我觉得用逐帧动画来做是最方便的，但是我解压了QQ的APK没有找到这个动画的素材，可能QQ是动态加载此类素材方便实时更新。所以没有办法，我们只能自己手动来画这个动画，因此细节上会有一定的差异，但是总体来说效果还是达到了。

我的整体思路是自定义一个view画出N个气泡，然后控制这些气泡随机向四周消散。好了，我们一步步来实现。

三、整合效果，在item进入界面时开启动画

我们观察实际效果是在特定用户进入界面时开启动画，因此我们也需要做这方面的处理。以ListView为例，我们知道ListView有一个方法是getLastVisiblePosition()，API里的解释是Returns the position within the adapter’s data set for the last item displayed on screen.（返回显示在屏幕里的适配器数据集中最后一项的位置）。因此我们只要利用这个方法在我们指定的那一项进入屏幕时开启动画即可。当然实际的情况可能更复杂，这里只是提供一个思路。毕竟路子有了，怎么走就看各位的手段了。
